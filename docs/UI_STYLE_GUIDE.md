# 第一阶段 UI 规范文档

本文件定义第一阶段 UI 必须遵守的视觉与交互规则。

---

## 一、视觉主方向

当前小程序 UI 必须延续已有风格：

1. 暖棕金配色
2. 米白到浅棕背景渐变
3. 大圆角卡片
4. 档案馆 / 收藏册式视觉语言
5. 胶囊主按钮 + 浅金次按钮

禁止做成：

1. 冷色科技风
2. 极简后台白底表格风
3. 玩家端和门店端两套完全割裂的设计

---

## 二、颜色建议

### 页面背景

- `#f6efe8` 到 `#ece0d2`

### 主按钮

- 深棕到铜色渐变
- `#6b3429 -> #a95f3b -> #d0a060`

### 卡片底色

- 暖白
- 浅米
- 浅杏

### 强调文字

- 棕红
- 铜金

### 错误提示

- 偏砖红
- 不用高饱和纯红

---

## 三、组件规则

### 卡片

- 主卡片圆角：`28rpx`
- 次级卡片圆角：`22rpx`
- 使用轻阴影，不做重悬浮感

### 按钮

- 主按钮：`button-primary`
- 次按钮：`button-secondary`
- 禁用按钮：需要明显弱化

### 输入框

- 圆角：`18rpx` 到 `22rpx`
- 背景：偏暖浅色
- 不做冷白输入框

---

## 四、文案语气

### 玩家端

- 偏沉浸、档案、组局、体验
- 避免后台术语
- 避免直接暴露底层错误码

### 门店端

- 可以更直接
- 但不能做成冰冷后台
- 重点是“清楚下一步要做什么”

---

## 五、页面级规则

### 首页

- 用 hero 卡片承接主题气质
- 搜索框不要做得像后台筛选器

### 大厅页

- 必须展示“待门店确认”
- 不得让用户误以为已经真实成团

### 队伍房间页

- 是沉淀页，不是聊天室
- 优先展示状态、成员、结果、集锦

### 工作台首页

- 优先展示待办
- 列表必须分组：
  - 待确认
  - 待开始
  - 进行中
  - 待上传集锦

### 场次管理页

- 必须清楚区分：
  - 确认成员
  - 开始场次
  - 结束场次
  - 自动结算
  - 上传集锦

---

## 六、禁止事项

1. 新增一套与现有小程序不一致的视觉风格
2. 在玩家端继续强化“手动打卡输入”为主流程
3. 用冷白表格页替代门店页
4. 一个页面里同时塞入多个业务域的大量复杂操作
