# 第一阶段 UI 流程规范

本文件定义第一阶段所有页面必须遵守的流程顺序和页面关系。

---

## 一、玩家流程

1. 首页
2. 主题详情
3. 组局大厅
4. 发起组局 / 加入组局
5. 待门店确认
6. 真实队伍房间
7. 结果页
8. 档案 / 徽章 / 排行榜

说明：

- 玩家主流程不再以“输入打卡码”为核心步骤
- 玩完后主要看到的是结果页和集锦入口

---

## 二、门店流程

1. 工作台入口
2. 首次授权
3. 工作台首页
4. 场次管理
5. 确认成员
6. 开始场次
7. 结束场次
8. 自动结算
9. 上传集锦

---

## 三、强制动作链

门店场次管理必须严格按以下顺序：

1. `确认成员`
2. `开始场次`
3. `结束场次`
4. `自动结算`
5. `上传集锦`

禁止：

1. 确认成员后直接默认开场
2. 开始和结束共用一个按钮
3. 结束后又要求玩家主流程手动打卡

---

## 四、页面展示边界

### 玩家端能看到

1. 主题
2. 活动
3. 大厅
4. 真实队伍房间
5. 结果与集锦
6. 档案、徽章、排行榜

### 玩家端不能看到

1. 门店工作台按钮链
2. 确认成员按钮
3. 开始场次按钮
4. 结束场次按钮

### 门店端能看到

1. 工作台首页
2. 待办分组
3. 场次管理按钮链
4. 集锦上传入口
5. 员工管理和授权码管理

---

## 五、关键状态映射

### 大厅卡片状态

- `招募中`
- `人数已满`
- `待门店确认`
- `确认成功`
- `异常取消`

### 房间状态

- `待确认`
- `待开场`
- `游戏中`
- `结算中`
- `已结算`
- `已归档`

### 场次管理主按钮状态

- `pending_confirm -> 确认成员`
- `ready -> 开始场次`
- `playing -> 结束场次`
- `settling -> 正在结算`
- `settled -> 上传集锦`
- `archived -> 无按钮`
