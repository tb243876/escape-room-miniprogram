# utils 目录职责

本目录按职责拆分，后续默认按此结构维护。

## 目录分层

- `cloudbase.js`
  - 对页面提供统一服务入口
  - 负责业务编排
  - 不承载大量底层细节实现

- `platform/`
  - 平台相关能力
  - 运行时配置、CloudBase 调用、本地存储适配都放这里

- `domain/`
  - 业务领域能力
  - 主题、档案、组局等规则、映射、字段整理放这里

## 维护约束

1. 页面层不要直接依赖 `platform/` 或 `domain/` 下的细节模块，优先通过 `cloudbase.js`
2. 如果某个业务逻辑开始明显膨胀，优先在 `domain/` 下新增模块，而不是继续堆到 `cloudbase.js`
3. 如果某个能力与微信运行时、存储、云环境强相关，优先放到 `platform/`
4. 不要重新把 `utils` 目录改回平铺式大杂烩
