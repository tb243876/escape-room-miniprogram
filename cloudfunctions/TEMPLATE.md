# 云函数模板说明

后续新增云函数时，默认按下面模板组织。

## 1. 目录模板

```text
cloudfunctions/<function-name>/
  index.js
  package.json
  <business>-domain.js
  response.js
  validate.js
```

## 2. index.js 模板职责

只做：

1. 读取 `event`
2. 读取用户身份
3. 校验参数
4. 开启事务
5. 查询状态
6. 调用领域规则
7. 落库
8. 返回统一结果

## 3. `<business>-domain.js` 模板职责

只放：

1. 领域默认对象
2. 字段归一化
3. 统计逻辑
4. 规则判断
5. 业务对象拼装

## 4. `response.js` 模板职责

只放：

1. 成功结果构造
2. 错误结果构造
3. 日志包装

## 5. `validate.js` 模板职责

只放：

1. 参数长度校验
2. 格式校验
3. 枚举值校验
4. 必填项校验

## 6. 当前项目要求

如果一个新云函数的 `index.js` 里开始出现以下内容，就应当考虑继续拆：

1. 大量常量规则
2. 多段复杂 if/else 业务判断
3. 重复返回结构
4. 重复数据库访问片段
5. 与其他云函数相似的默认对象构造
